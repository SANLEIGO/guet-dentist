# 口腔内窥镜智能影像分析与三维重建系统项目书

## 1. 项目名称

口腔内窥镜智能影像分析与三维重建系统

## 2. 项目定位

本项目面向口腔检查、病例记录、患者沟通和辅助诊疗场景，建设一套以口腔内窥镜影像为核心输入的智能化系统。系统以 2D 口腔图像采集为起点，完成牙齿区域识别、健康状态辅助判断、全口 2D 可读图像生成，并逐步扩展至牙弓三维重建，最终形成覆盖移动端、PC 端和服务器端的一体化解决方案。

本项目不是单一算法验证，而是集算法、前后端产品、数据管理与服务端支撑于一体的完整系统工程。

## 3. 项目背景与意义

传统口腔检查依赖医生现场观察、局部照片记录和患者主观理解，存在以下问题：

- 口腔局部照片分散，缺乏完整牙弓级别的连续展示。
- 不同拍摄角度和光照条件导致图像难以直接对比与归档。
- 患者难以直观理解病灶位置、范围和变化过程。
- 基层场景设备有限，高成本影像设备不易普及。
- 后续三维重建和长期随访缺少标准化影像输入基础。

口腔内窥镜具有成本低、便携性强、适合基层和家庭场景的优势。如果能通过算法和系统工程能力，把分散的 2D 口腔图像转化为结构化、连续化、可解释的诊疗信息，就能够显著提升口腔检查、病例管理和医患沟通效率，并为远程医疗和数字化口腔建模提供基础。

因此，本项目具有以下价值：

- 提升口腔图像采集结果的可读性和标准化程度。
- 为医生提供更完整的影像辅助信息。
- 为患者提供更直观的口腔健康展示方式。
- 为后续三维建模、智能分析和远程会诊建立统一数据底座。

## 4. 项目目标

### 4.1 总体目标

构建一套基于口腔内窥镜图像的智能影像分析系统，实现从图像采集、预处理、拼接、病灶分析到三维重建的整体技术链路，并形成可部署到手机端、PC 端和服务器端的产品化系统。

### 4.2 分阶段目标

#### 第一阶段：2D 图像理解与拼接

- 实现多张口腔内窥镜图像上传、管理与可视化。
- 实现牙齿区域分割，排除软组织与无关背景干扰。
- 实现相邻图像特征提取、匹配、几何配准与图像融合。
- 生成可供医生和患者阅读的 2D 连续口腔图像。
- 输出基础质量评估和诊断辅助信息。

#### 第二阶段：健康状态智能辅助判断

- 基于口腔图像开展龋坏、牙结石、牙龈炎症等目标的识别研究。
- 建立面向病灶区域的检测、分类或分割模型。
- 形成可视化风险提示、区域标注和辅助判读结果。

#### 第三阶段：三维重建与系统集成

- 基于多视角 2D 图像恢复牙弓或局部牙列的三维结构。
- 探索 2D 到 3D 的建模方法和数据表达方式。
- 打通移动端采集、PC 端查看、服务器端管理和模型服务。
- 建设完整的数据流、用户流和管理流。

## 5. 建设内容

本项目计划建设以下四个层面的能力。

### 5.1 移动端应用

移动端主要承担图像采集和即时反馈功能，支持 Android 和 iOS。

主要功能包括：

- 口腔内窥镜图像采集与上传。
- 拍摄引导与重叠区域提示。
- 采集质量初筛，如模糊、曝光异常、视野不足。
- 患者基本信息关联。
- 结果查看与历史记录调阅。

### 5.2 PC 端应用

PC 端主要面向医生、研究人员和管理人员，承担高可读性结果展示与精细分析功能。

主要功能包括：

- 牙齿区域分割结果展示。
- 2D 拼接图生成与下载。
- 特征匹配和配准过程可视化。
- 诊断辅助结果展示。
- 病例归档、导出与报告查看。

### 5.3 服务器端与管理后台

服务器端负责模型服务、任务调度、数据存储和权限管理。

主要功能包括：

- 用户、病例和图像数据管理。
- 算法模型统一部署与版本控制。
- 推理任务调度与结果回传。
- 标注数据管理与模型训练支持。
- 系统日志、审计和权限控制。

### 5.4 核心算法引擎

算法引擎是系统核心，围绕口腔图像处理和智能分析展开。

主要能力包括：

- 图像预处理与质量增强。
- 牙齿区域检测与分割。
- 图像特征提取与匹配。
- 几何配准与全景拼接。
- 图像融合与缝隙弱化。
- 口腔健康状态辅助识别。
- 三维重建与结构表达。

## 6. 技术路线

本项目采用“传统计算机视觉 + 深度学习 + 系统工程”的复合技术路线。

### 6.1 图像采集与预处理

口腔内窥镜图像通常存在视角变化大、闪光灯反光、亮度不均、软组织遮挡和背景复杂等问题，因此需先进行标准化处理：

- 图像格式解码与尺寸归一化。
- 色彩空间变换与亮度增强。
- 去噪、对比度增强与清晰度评估。
- 曝光异常和模糊图像筛查。

### 6.2 牙齿区域检测与分割

为了减少唇、颊、舌等区域对后续拼接和识别的干扰，系统先定位牙齿主体区域。

可采用的方法包括：

- 基于 YOLO 的牙齿目标检测或实例分割。
- 基于 CNN 的牙齿区域分类与边界细化。
- 基于 OpenCV 的颜色、纹理和图像分割后处理。

当前原型已采用 AlphaDent 与 OpenCV 后处理结合的方式，具备牙齿区域分割基础能力。

### 6.3 图像拼接

拼接模块用于把多张局部口腔图像重建为连续 2D 口腔图。

关键步骤包括：

- 在牙齿区域内提取稳定特征。
- 进行相邻图像特征匹配与异常匹配剔除。
- 估计单应性矩阵或仿射变换。
- 对变换结果进行图像融合和边界平滑。
- 输出拼接图、匹配可视化和质量指标。

当前原型使用 OpenCV 实现了 ORB、AKAZE、SIFT 等特征方法，并支持基于 RANSAC 的几何配准。

### 6.4 健康状态辅助分析

系统后续将围绕以下任务展开研究：

- 龋坏区域检测。
- 牙结石与菌斑识别。
- 牙龈红肿、出血相关区域判别。
- 病灶风险评分与变化趋势对比。

可采用的技术包括：

- YOLO 系列目标检测模型。
- CNN 分类网络。
- 分割网络和病灶区域热力图方法。

### 6.5 2D 到 3D 重建

在生成高质量 2D 连续图像基础上，进一步探索多视角牙列三维重建。

研究方向包括：

- 基于多视图几何的三维恢复。
- 基于深度学习的单目或多目三维重建。
- 点云、网格或参数化牙弓模型表示。
- 2D 语义信息向 3D 结构空间映射。

## 7. 系统总体架构

系统总体上分为采集层、算法层、服务层和应用层。

### 7.1 架构说明

#### 采集层

- 口腔内窥镜设备
- 手机端采集应用
- PC 端图像导入接口

#### 算法层

- 图像预处理模块
- 牙齿分割模块
- 图像拼接模块
- 健康分析模块
- 三维重建模块

#### 服务层

- 模型推理服务
- 任务调度服务
- 数据存储服务
- 用户与病例管理服务
- 日志与权限服务

#### 应用层

- 医生端查看与分析界面
- 患者端结果查看界面
- 管理后台

### 7.2 数据流

1. 用户通过口腔内窥镜采集多张口腔图像。
2. 图像上传至移动端或 PC 端。
3. 图像传入服务器或本地算法引擎进行预处理。
4. 执行牙齿分割、拼接与质量评估。
5. 输出 2D 连续图像和辅助诊断结果。
6. 在满足条件时进入三维建模流程。
7. 结果归档并支持历史病例调阅。

## 10. 创新点

### 10.1 低成本口腔影像数字化

以普通口腔内窥镜为采集设备，降低高端成像设备依赖，提升基层可用性。

### 10.2 面向牙科场景的专用拼接链路

针对口腔图像反光强、软组织干扰多、纹理重复高等特点，构建牙齿区域约束下的拼接方法。

### 10.3 从单点算法走向完整系统

项目覆盖采集端、算法端、展示端和服务端，强调系统集成能力而不是单一模型性能。

### 10.4 为三维重建建立标准化入口

通过连续 2D 影像和结构化分割结果，为后续三维建模和数字口腔表达提供稳定输入。

## 15. 风险与应对

### 15.1 图像质量波动风险

风险：采集设备、光照、角度差异较大，影响分割和拼接稳定性。

应对：增加采集规范、质量筛查和鲁棒预处理机制。

### 15.2 病灶识别数据不足

风险：高质量标注数据获取成本高，影响模型训练效果。

应对：建立标注规范，逐步积累病例数据，并采用迁移学习和半监督策略。

### 15.3 多端系统集成复杂

风险：移动端、PC 端和服务器端开发节奏不同，系统联调成本高。

应对：采用模块化架构，先定义统一接口和数据格式，再分阶段集成。

### 15.4 三维重建难度较高

风险：2D 到 3D 的重建效果受采集质量和模型能力影响较大。

应对：先完成高质量 2D 连续图像生成，再逐步推进局部三维重建验证。

## 16. 结论

本项目聚焦口腔内窥镜场景，目标不是单点算法优化，而是建设一套覆盖图像采集、智能分析、结果展示和三维扩展的完整系统。其核心价值在于利用低成本设备获取高可读性的口腔数字影像，并在此基础上叠加病灶识别、病例管理和三维建模能力。

从当前进展看，项目已经具备牙齿分割与 2D 拼接的原型基础，具备继续向多图拼接、辅助诊断、多端协同和三维重建方向推进的现实可行性。后续通过完善算法稳定性、补齐系统架构和加强数据支撑，可逐步形成具备研究价值和应用潜力的口腔智能影像平台。
