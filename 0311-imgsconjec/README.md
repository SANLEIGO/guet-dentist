# Oral Image Stitcher

用于口腔内窥镜二维图像在不同光照条件下的上牙/下牙局部拼接，采用最容易上手的 Web 方式：`Streamlit`。

## 🎉 新功能：改进拼接算法

本项目现已包含改进的拼接算法，提供更好的医学影像质量：

### 主要改进
- **多频段融合**：使用拉普拉斯金字塔消除拼接缝
- **增强预处理**：自适应牙齿区域检测和增强
- **光照鲁棒性**：对不同光照条件有更好的适应性
- **医学影像质量**：满足医学诊断要求

### 使用方法
在Web界面侧边栏选择"改进算法（推荐）"即可使用新的拼接算法。

详细说明请查看 [IMPROVEMENTS.md](IMPROVEMENTS.md) 和 [COMPLETION_SUMMARY.md](COMPLETION_SUMMARY.md)。

## 原有功能

当前流程按 `上牙/下牙 + 左侧段/右侧段/完整牙弓` 组织。半侧牙弓和完整牙弓使用不同的拼接策略。

## 方法选择

**默认优先使用改进算法**：

- 多频段融合消除不同光照条件的拼接缝
- 自适应牙齿区域检测和增强
- SIFT特征检测更稳定
- 多级备选方案提高鲁棒性

如果遇到兼容性问题，可选择**原始算法**：

- 默认使用 `LoFTR` 深度匹配器：
  - 对弱纹理、重复纹理和亮度变化的鲁棒性通常优于传统局部特征。
  - 对牙齿表面这种高反光、边缘重复的场景更稳定。
  - 若环境缺少 `torch/kornia`，程序会自动回退到 `AKAZE + RANSAC`，保证基础可用。

基准图不是手工固定，而是根据当前导入图像自动选择：

- 匹配图连通性
- RANSAC 内点数
- 图像清晰度
- 曝光适中程度
- 上传顺序邻近性

这比硬编码为“第一张图”更适合普通内窥镜拍摄数据。

当前版本还会优先采用“链式拼接”思路：

- 更优先信任上传顺序相邻的图像对
- 对顺序距离较远的图像对降低连接权重
- 更适合半侧牙弓按连续视角拍摄的常见采集方式
- 在“完整牙弓”模式下，会优先让中央图作为桥接基准，并对相邻图对启用更宽松的桥接匹配
- 在“完整牙弓”模式下，默认按上传顺序走严格链式拼接，因此上传顺序必须尽量符合从一侧到另一侧的连续采集顺序
- 匹配前会压低黑边、软组织和高光区域的影响，并检查内点的空间分布，减少局部误匹配拖偏整图
- 在“完整牙弓”模式下，会先做整组主方向一致性校正，并输出更保守、更可读的序列条带展开图，而不是自由形态全景图

## 运行

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

## 使用方式

1. 在左侧选择待拼接区域：`上牙` 或 `下牙`。
2. 再选择待拼接范围：`左侧段`、`右侧段` 或 `完整牙弓`。
3. 上传同一牙弓、同一范围的一组图片。
4. 查看预检结果，程序会先按清晰度和曝光给出建议剔除项。
5. 查看基准图候选评分，程序会解释为什么推荐某张图作为拼接基准。
6. 可手动勾选最终参与拼接的图像，并可选择继续自动选基准图，或手动指定基准图。
7. 点击“开始拼接”，获得结果、日志与下载按钮。
8. 可查看与基准图之间的匹配连线诊断，并下载完整 JSON 诊断报告。

## 说明

- 当前版本支持“上牙/下牙 + 左右侧段 + 完整牙弓”。
- 更推荐导入同一段牙弓、相邻视角、重叠区域充分的图像。
- 支持手动筛图、手动指定基准图、查看与基准图的匹配可视化。
- 如果后续你要继续扩展，可加入牙列分区、自动筛除模糊图、以及分段弧形配准。
