# 牙齿图像拼接算法改进说明

## 概述

本项目针对口腔内窥镜拍摄的不同光照条件下的牙齿图像拼接问题，提供了改进的拼接算法，以获得更好的医学影像质量和人眼可读性。

## 主要改进

### 1. 多频段融合（Multi-band Blending）

**问题**：原有的简单alpha混合在不同光照条件下会产生明显的拼接缝。

**解决方案**：
- 实现拉普拉斯金字塔融合
- 在多个频率层次上分别融合图像
- 高频部分保留细节，低频部分平滑过渡
- 消除光照差异带来的拼接缝

### 2. 改进的图像预处理

**特点**：
- 自适应牙齿区域检测
- 基于HSV色彩空间的牙齿掩膜提取
- 排除软组织（牙龈）的干扰
- CLAHE对比度受限自适应直方图均衡化
- 双边滤波降噪

### 3. 增强的特征检测

**改进**：
- 使用SIFT作为主要特征检测器（比AKAZE更稳定）
- ORB作为备用特征检测器
- 更多的特征点数量（3000个SIFT特征点）
- 自适应特征检测参数

### 4. 更鲁棒的特征匹配

**特点**：
- Lowe's ratio test筛选优质匹配
- RANSAC估计单应性矩阵
- 仿射变换作为备选方案
- 更严格的内点验证

### 5. 自动质量评估

**功能**：
- 图像清晰度评分（基于拉普拉斯方差）
- 曝光质量评估
- 自动筛选低质量图像
- 智能基准图选择

### 6. 改进的几何校正

**特点**：
- 单应性矩阵投影变换
- 自动画布大小计算
- 有效区域裁剪
- 边缘羽化处理

## 技术细节

### 预处理流程

1. **色彩空间转换**：BGR → LAB
2. **牙齿区域提取**：
   - 背景分离（灰度阈值）
   - 牙齿识别（HSV范围）
   - 软组织排除（HSV粉色检测）
3. **对比度增强**：CLAHE
4. **降噪处理**：双边滤波

### 特征检测与匹配

1. **特征检测**：
   - SIFT: 3000特征点，8层金字塔
   - ORB: 2000特征点作为备用
2. **特征匹配**：
   - KNN匹配（k=2）
   - Lowe's ratio test（0.75阈值）
   - RANSAC几何验证

### 多频段融合

1. **金字塔构建**：
   - 6层拉普拉斯金字塔
   - 高斯金字塔用于权重
2. **分层融合**：
   - 每层独立融合
   - 权重归一化
3. **图像重建**：
   - 从粗到细重建
   - 裁剪有效区域

## 使用方法

### 基本使用

```bash
# 安装依赖
pip install -r requirements.txt

# 启动应用
streamlit run app.py
```

### 算法选择

在Web界面中可以选择：
- **改进算法（推荐）**：使用多频段融合和增强预处理
- **原始算法**：使用基础的拼接方法

### 参数调整

如需调整融合参数，可修改 `enhanced_stitching.py` 中的：
- `feather_radius`: 羽化半径（默认30）
- `levels`: 金字塔层数（默认6）

## 注意事项

1. **图像采集**：
   - 保持相同拍摄距离
   - 相邻图像有足够重叠（30%以上）
   - 避免过度模糊

2. **拼接限制**：
   - 一次只拼接同一牙弓（上牙或下牙）
   - 建议分侧段拼接（左侧段、右侧段）
   - 完整牙弓需要按顺序拍摄

3. **性能考虑**：
   - 多频段融合计算量较大
   - 大图像可能需要较长处理时间
   - 建议图像分辨率不超过2000x2000
