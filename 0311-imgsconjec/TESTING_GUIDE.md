# 测试指南

## 如何测试改进的拼接算法

### 1. 启动应用程序

```bash
# 确保在项目目录中
cd /Users/administrator/Downloads/teeth/0311-imgsconjec

# 激活虚拟环境（如果有）
source .venv/bin/activate

# 安装/更新依赖
pip install -r requirements.txt

# 启动 Streamlit 应用
streamlit run app.py
```

### 2. 使用改进算法

1. 在 Web 界面左侧的 **"输入设置"** 面板中：
   - 选择拼接区域：**"上牙"** 或 **"下牙"**
   - 选择拼接范围：**"左侧段"**、**"右侧段"** 或 **"完整牙弓"**
   - **重要**：在 **"拼接算法"** 下拉菜单中选择 **"改进算法（推荐）"**

2. 上传同一区域的口腔图像（支持多选）

3. 查看预检结果：
   - 检查图像质量评分
   - 确认哪些图像被建议剔除

4. 手动确认参与拼接的图像（可选）

5. 查看基准图候选评分：
   - 系统会显示每张图像的评分
   - 推荐的基准图会被标记

6. 选择基准图模式：
   - **"自动选择"**：使用系统推荐的基准图
   - **"手动指定"**：自己选择基准图

7. 点击 **"开始拼接"** 按钮

### 5. 故障排除

如果遇到问题：

#### 多频段融合失败
- **日志显示**：`多频段融合失败，使用简单融合`
- **原因**：多频段融合遇到错误，自动回退到简单融合
- **结果**：仍会生成拼接结果，质量可能略低

#### 特征检测失败
- **日志显示**：`检测到 0 个特征点`
- **原因**：图像质量太差或内容不合适
- **解决**：尝试上传更清晰的图像

#### 匹配失败
- **日志显示**：`匹配到 0 对特征点`
- **原因**：图像之间没有足够的重叠区域
- **解决**：确保相邻图像有 30% 以上的重叠

### 6. 性能优化建议

#### 图像准备
- 分辨率：建议不超过 2000x2000
- 格式：JPG、PNG 均可
- 质量：避免过度模糊的图像

#### 拍摄建议
- 保持相同拍摄距离
- 相邻图像重叠 30% 以上
- 避免过度曝光或欠曝
- 尽量保持相同的光照条件

#### 处理建议
- 一次只拼接同一牙弓（上牙或下牙）
- 建议分侧段拼接（左侧段、右侧段）
- 完整牙弓需要按顺序拍摄

### 7. 对比测试

您可以对比两种算法的效果：

1. **改进算法（推荐）**：
   - 多频段融合
   - 更好的光照处理
   - 减少拼接缝

2. **原始算法**：
   - 基础融合方法
   - 处理速度较快
   - 效果相对简单

### 8. 常见问题

**Q: 为什么多频段融合失败了？**
A: 可能是图像之间的变换过于复杂，系统会自动使用简单融合作为备选。

**Q: 拼接结果有明显的拼接缝怎么办？**
A: 尝试：
   - 确保选择了"改进算法"
   - 检查图像是否有足够的重叠
   - 改善图像质量

**Q: 处理时间很长正常吗？**
A: 改进算法使用多频段融合，计算量较大，处理时间会比原始算法长一些，这是正常的。

**Q: 能处理多少张图像？**
A: 建议一次处理 2-8 张图像。过多图像可能导致处理时间过长。
